Low Price Domains

Redesigning a Domains Website

Role: UX/UI Designer & Developer

LowPriceDomains (LPD) was outdated like many domain/hosting websites out there. My job was to redesign and subsequently integrate a new version of the site. At first glance the problem was obvious, the website was old and therefore unappealing.

With many websites offering domain and hosting services “we can only hope to make a difference visually” I thought. However digging a little deeper, I discovered many web and mobile apps focusing entirely on helping users find an ideal domain name. GoDaddy for example built an application dedicated exclusively at finding domain names. All the big domain companies have name suggestion and filtering to some degree, so I figured I should somehow follow that idea in my design.

Going Flat

I opted for a flat UI. This I felt would allow me to concentrate on thinning out the main navigation and make each page more understandable. I worked on the navigation hierarchy as I concluded that having a three-row navigation was too confusing. Logging in before would take the user to a subdomain, which proved to be somewhat confusing after a few tries. I decided to have a modal to log in and avoid going into the subdomain until the user is ready to check out.

Jumping right to Search

This is self-explanatory. LPD offers SSL certificates but its main selling items are domain names. Having therefore a domain search box right on the home page is more practical consequently more beneficial for both the user and LPD.

Domain Search Engine

For now LPD uses WHMCS as its prime domain search engine. I worked though on a separate project that would allow for gtlds as well as domain name suggestion to take place right on the search engine. This project required the integration of APIs, a lot more front-end work and perhaps cutting out WHMCS altogether to use a more compatible cart. These are not design challenges, but they did halt the production of this site.
